

# 웹 기반의 클라이언트/서버 프로그램


웹 요청에 대해 웹 문서를 만들기 위해서는...
웹 서버가 요청에 맞는 웹 문서를 돌려줄 것이다.

하지만 어떠한 요청에 대한 문서가 미리 만들어 지는 것은 불가능하다.
그렇기 때문에, 이를 만들어 줄 수 있게 코드가 들어간다. 
이 코드를 돌려 웹 문서를 만들어 주기 위해서는(문서화) 추가적인 무언가가 필요하다.

이것이 바로 'WAS(Web Application Server)'라고 한다. 

1. 동적으로 문서를 만들기 위한 코드 ==> Server Application
2. 코드를 실행할 수 있게 해주는 환경 ==> Web Server + WAS 



# 왜 웹 서버 응용 프로그램을 Servlet이라 명칭할까?


1. GET: list/reg/edit/del 을 요청(CRUD 요청 가능)
2. 웹 서버는 del이라는 것를 찾을 것이다.
3. 코드를 문서로 바꾸기 위해 자바 서버 어플리케이션이 실행되고 끝난다. 
--> del이라는 코드는 사용자가 무엇을 요청하는가에 따라서
    달라야 하기 때문에 요청에 따라 파편화 되어 있을 것이다.

===> Server Application이 파편화 되었다.
     Server Application Let(조각나다) = Serv + Let = Servlet 탄생
