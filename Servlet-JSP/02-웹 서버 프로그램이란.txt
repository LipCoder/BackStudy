
# 일반적인 업무용 프로그램

예전에는 서버 프로그램이라는게 없었음
그저 프로그램만 있었고, 껍데기에 따라 GUI프로그램, 콘솔 프로그램으로 나뉨

# CS(클라이언트/서버) 프로그램
서버/클라이언트 구조로 각자 역할이 정해지면서 
어떠한 서비스를 서버에 요청하면 서버가 제공해주는 방식으로 변화
(인터넷이 발달하면서 더더욱 발전되었다.)

요청 --> 서버 --> DB --> 서버 --> 결과


요청자(클라이언트), 제공자(서버)


* 클라이언트 프로그램의 문제 --> 업데이트에 대한 배포가 어렵다.
> 설치도 해야하며 설치시 플랫폼 환경에 따라 충돌하는 문제도 생겼음..
==> 이를 최소화 하고자, 웹 기반 프로그램이 각광받게 되었다.


데이터 전송 방법은 여러가지(소켓, RPC, ...)가 있지만,
웹은 HTTP를 사용한다. 


# 데이터가 아닌 문서를 전달하는 웹을 사용하면?
브라우저 <--(Response)----------------(GET Request)--> 웹서버 

역동적인 기능(게임 등)을 하는 프로그램이 아닌 이상 왠만하면 웹 프로그램으로 구성할 수 있다.

웹 서버 + (웹 프로그램 ==> 동적 문서를 만들 수 있다.)
웹 서버는 그냥 문서등의 자료만 제공 (정적임)


예전에는 웹 프로그램 전반 서버 프로그래머가 다 구성했었지만,
자바 스크립트의 등장으로 웹 서버에서는 데이터만 요청 하고(백엔드), 
자바 스크립트로 웹 프로그램을 만드는 시대가 도래하였다.(프론트엔드)
