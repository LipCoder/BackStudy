# Database와 DBMS의 개요

- 중복을 없애는 방법으로 결함을 줄인다.
    ex) 대학교에서 각 (부서의) 컴퓨터에 정보를 관리한다고 가정하자...
        만약에 대학교 등록금을 납입을 한 학생이 도서관을 이용하려고 한다.
        그러면 도서관에서는 이 사람이 재학생인지 확인하기 위해서 
        납입여부에 대한 데이터가 필요할 것이다.
        근데 도서관 데이터에서 납입에 대한 데이터가 반영이 안되어 있다면????
        그 학생은 이 대학교의 재학생이 아니게 되어 도서관을 이용할 수 없을 것이다.

        ==> 데이터를 각자 가지고 있는 것 보다는... 
            한 곳에 모아 관리한다면 이런 문제를 해결할 수 있지 않을까?



- 중복을 없애기 위하여 데이터를 관리하는 상자(테이블)이 여기저기 찢어졌다.
   
    문제점 1: 이것을 참조하여 이용하는 방법이 예전보다 더 까다로워졌다...
              그러면 이걸 어떻게 해야 쉽게 데이터를 사용할 수 있을까?
    
    문제점 2: 모두가 한 곳에서 쓰다보니 동시성 문제가 발생한다.
              이것을 관리할 필요성이 있어졌다.

    문제점 3: 모두가 같이 쓰다보니 성능적인 문제가 생겼다.
              이것 또한 고민이 필요
    
    문제점 4: 보안 문제

    ==> 이러한 문제점들을 관리해주는 프로그램이 필요하겠다..
        관리자를 통해 부탁해서 데이터를 써야 되겠다..


 * 방법론 종류 탄생
        Hierarchical DBMS
        Network DBMS
        Object-Oriented DBMS
        Relational DBMS ==> 대세


- DBMS 프로그램의 필요성을 바탕으로 이를 개발하면서 3개의 유명 DBMS 회사가 탄생하였다.
    1. MS SQL Server
    2. MySQL
    3. Oracle  ==> 이걸 쓸거다.


- DBMS에게 명령하는법? SQL!
    SQL(Structured Query Language)
    DDL (데이터를 정의한다)
    DML (데이터를 조작한다)
    DCL (데이터를 제어한다)
